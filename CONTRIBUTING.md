# Contributing to MolecularFingerprints.jl

We welcome contributions to MolecularFingerprints.jl! Whether you're fixing a bug, adding a new feature, or improving documentation, your help is appreciated. Please follow the guidelines below to ensure a smooth contribution process.

## How to Contribute (core team members)
1. Clone the repository to your local machine.
2. Create an issue or select an existing one to work on.
3. Create a new branch for your feature or bug fix, you should name it descriptively (e.g., `feature/new-fingerprint-algorithm` or `bugfix/fix-similarity-calculation`).
4. Make your changes and commit them with clear and descriptive messages.
5. Push your changes to the repository.
6. Open a pull request against the `main` branch.
7. Ensure that all tests pass and that your code adheres to the project's coding standards.

## How to Contribute (outside collaborators)
1. Fork the repository on GitHub.
2. Clone your fork to your local machine.
3. Create a new branch for your feature or bug fix.
4. Make your changes and commit them with clear and descriptive messages.
5. Push your changes to your fork on GitHub.
6. Open a pull request against the main repository's `main` branch.
7. Ensure that all tests pass and that your code adheres to the project's coding standards.

## Code Style
Please follow the Julia style guide for consistency. You can use tools like `JuliaFormatter.jl` to automatically format your code.

## Commit Messages

Follow the Conventional Commits specification:
https://www.conventionalcommits.org/en/v1.0.0/#specification

If you're not familiar with Conventional Commits, please refer to this cheatsheet for guidance on commit types:
https://www.bavaga.com/blog/2025/01/27/my-ultimate-conventional-commit-types-cheatsheet/
